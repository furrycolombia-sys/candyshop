### DRY-001: Duplicate URL search param parsers instead of using centralized `catalogSearchParams`

**Files:**

- `apps/store/src/features/products/presentation/components/CategoryFilter.tsx` (line 17-20)
- `apps/store/src/features/products/presentation/components/TypeFilter.tsx` (line 18)
- `apps/store/src/features/products/presentation/components/SearchBar.tsx` (line 13)

**Description:** The project defines a centralized `catalogSearchParams` object in `features/products/domain/searchParams.ts`, but `CategoryFilter`, `TypeFilter`, and `SearchBar` each create their own standalone `useQueryState` with `parseAsString.withDefault("")` instead of consuming parsers from the centralized definition. This violates the DRY principle and the [URL State Rule](../../.claude/rules/url-state.md) which says parsers belong in `domain/searchParams.ts` and hooks should import from there.

**Fix:** Import the individual parsers from `catalogSearchParams` or use `useQueryStates(catalogSearchParams)` in each filter component, matching what `ProductCatalogPage` already does. At minimum, each component should reference the centralized parser:

```typescript
// CategoryFilter.tsx
import { catalogSearchParams } from "@/features/products/domain/searchParams";
const [category, setCategory] = useQueryState(
  "category",
  catalogSearchParams.category,
);
```

**Files:**

- `apps/store/src/features/products/presentation/components/product-detail/StarRow.tsx`

Remove `RatingStars.tsx` and update `HeroSection.tsx` to use `StarRow` with a custom `filledClassName` derived from `theme.text`.

### BUG-001: `SearchBar` debounce timer not cleaned up on unmount

**File:** `apps/store/src/features/products/presentation/components/SearchBar.tsx` (lines 15, 33-36)

**Description:** The `timerRef` stores a `setTimeout` handle but there is no `useEffect` cleanup to cancel the pending timeout on unmount. If the user types, navigates away before the 300ms debounce fires, the callback will attempt to call `setQuery` on an unmounted component. While React 18+ suppresses the warning, the state update is still wasteful and could cause unexpected URL changes.

**Fix:** Add cleanup:

```typescript
useEffect(() => {
  return () => {
    if (timerRef.current !== null) {
      clearTimeout(timerRef.current);
    }
  };
}, []);
```

Or better yet, use the `useDebounce` hook from the `shared` package or `usehooks-ts` to avoid manual timer management, per the [Libraries Over Manual Code](../../.claude/rules/libraries-over-manual-code.md) rule.

**Fix:** Create a shared `formatPrice(amount: number, currency: string)` utility that uses `Intl.NumberFormat` for locale-aware currency formatting:

}

### PERF-003: `useAddToCart` has `product` in dependency array causing unnecessary re-creations

**File:** `apps/store/src/shared/application/hooks/useAddToCart.ts` (line 67)

**Description:** The `handleAddToCart` callback depends on `[addItem, product, flyCtx]`. Since `product` is an object prop, a new reference on every parent render will recreate the callback. This could cause unnecessary re-renders of child components receiving `handleAddToCart`.

**Fix:** Destructure the needed product fields and use them as individual dependencies, or use a ref for the product:

```typescript
const productRef = useRef(product);
productRef.current = product;

const handleAddToCart = useCallback(
  (e: React.MouseEvent<HTMLButtonElement>) => {
    const p = productRef.current;
    // use p instead of product
  },
  [addItem, flyCtx],
);
```
