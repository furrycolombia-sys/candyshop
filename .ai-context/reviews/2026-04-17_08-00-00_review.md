## Pattern Discovery

### Good Patterns — Keep and Enforce

| ID      | Pattern                                                                                                                                                                                                       | Status                                            |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| PAT-001 | **Permission Guard + PageContent Split** — Every protected page is split into `*Page` (permission check) and `*PageContent` (UI). Guards return null while loading and render `AccessDeniedState` on failure. | Consistent across 8+ pages                        |
| PAT-002 | **nuqs Domain Search Params** — URL state parsers are defined in `domain/searchParams.ts`, cleanly placing URL shape definition in the domain layer.                                                          | Consistent across features with URL state         |
| PAT-003 | **Feature Query Key Constants in domain/constants.ts** — Every TanStack Query key is declared as a constant in the feature's `domain/constants.ts`.                                                           | Consistent across 9+ features                     |
| PAT-004 | **Domain Constants Unit Tests** — Every `domain/constants.ts` has a matching `constants.test.ts` that pins the exact string values of query keys.                                                             | Consistent across 9 features                      |
| PAT-005 | **Infrastructure Query Tests** — Infrastructure query files are tested using either MSW (audit feature) or chainable in-memory mocks.                                                                         | Two variants exist — see INC-001                  |
| PAT-006 | **Application Hook Tests use createWrapper() + renderHook** — Hook tests create a local `createWrapper()` with `retry: false`, then use `renderHook + waitFor`.                                               | Highly consistent across 24+ test files           |
| PAT-007 | **Supabase ESLint Disable Comment Pattern** — All hooks excluding the supabase client from `queryKey` add a standardized explanatory comment.                                                                 | Consistent across 16 files                        |
| PAT-008 | **Permission-Aware PageContent Props** — When a page checks multiple permissions, it passes results as boolean props (`canCreate`, `canUpdate`, `canDelete`) to the `PageContent` component.                  | Consistent where multiple permission checks apply |

### Inconsistencies — Should Be Addressed

| ID      | Inconsistency                                                                                                                                                                                                                                                                                                               | Recommendation                                                                                             |
| ------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| INC-001 | **MSW vs chainable mock for infrastructure tests** — Audit feature uses MSW HTTP-level interception; all other features use chainable in-memory Supabase mocks. MSW is more robust as it validates actual URL construction.                                                                                                 | Prefer MSW for new infrastructure tests. Document the preference.                                          |
| INC-002 | **Supabase client access: useSupabase() vs createBrowserSupabaseClient() inline** — Most hooks use `useSupabase()` (singleton). Three files create the client directly, including `apps/store/src/features/products/infrastructure/productQueries.ts` which creates a new client per call, bypassing the singleton pattern. | Standardize on `useSupabase()` in hooks. Infrastructure functions should accept the client as a parameter. |
| INC-003 | **MSW server setup missing in studio** — `admin`, `payments`, and `store` configure MSW in `test/setup.ts`. Studio's `test/setup.ts` only runs `cleanup()` — no MSW server configured. Future HTTP-level tests in studio would silently not intercept requests.                                                             | Add MSW server setup to studio's `test/setup.ts`.                                                          |

### Anti-Patterns — Should Be Fixed

| ID      | Anti-Pattern                                                                                                                                                                                | Recommendation                                                                                                                                                   |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| PAT-012 | **createWrapper() boilerplate repeated in 24+ test files** — The `createWrapper()` function is defined inline in every hook test file instead of being shared from a test utilities module. | Extract `createWrapper()` into `apps/[app]/src/test/utils.tsx` and export it. Saves ~7 lines per test file and ensures consistent default `QueryClient` options. |

### Rule Candidates

| ID       | Suggested Rule                                                        | Rationale                                                                                                                                                                                                                                                             |
| -------- | --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RULE-001 | **Infrastructure Functions Must Accept Supabase Client as Parameter** | All infrastructure functions currently accept the Supabase client as a parameter except `apps/store/src/features/products/infrastructure/productQueries.ts`. Codify this: infrastructure functions MUST accept the client as a parameter, never create it internally. |
| RULE-002 | **Prefer MSW over Chainable Mocks for Infrastructure Tests**          | MSW HTTP-level tests catch URL construction bugs that chainable mocks cannot. Document that new infrastructure tests should use MSW with `server.use()` overrides.                                                                                                    |
| RULE-003 | **App-Level Test Wrapper Utility**                                    | `createWrapper()` is repeated in 24+ files. Each app should export a `createWrapper`/`renderHookWithQuery` utility from `src/test/utils.tsx`.                                                                                                                         |
| RULE-004 | **All Apps Must Configure MSW in test/setup.ts**                      | Even if current tests don't use HTTP, all apps should configure `server.listen/resetHandlers/close` in `setup.ts` so new tests work correctly without extra setup.                                                                                                    |
