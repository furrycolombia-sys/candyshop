# Fix Full Review Skill

> Interactively fix issues found in code review reports.

---

## Description

The `/fix-full-review` skill reads a code review report generated by `/full-review` and helps you fix issues interactively. Each issue is presented with context, and you confirm before any changes are made.

---

## Usage

```
/fix-full-review [--report <path>] [--severity <levels>] [--category <categories>] [--dry-run]
```

Or natural language:

- "Fix the review issues"
- "Fix critical bugs from the last review"
- "Address SOLID violations"

---

## Parameters

| Parameter    | Required | Default | Description                                  |
| ------------ | -------- | ------- | -------------------------------------------- |
| `--report`   | No       | latest  | Path to review report (default: most recent) |
| `--severity` | No       | all     | Filter: `critical`, `warning`, `suggestion`  |
| `--category` | No       | all     | Filter: `arch`, `solid`, `bug`, etc.         |
| `--dry-run`  | No       | false   | Preview fixes without applying them          |
| `--auto`     | No       | false   | Auto-confirm all fixes (use with caution)    |

---

## Steps

### 1. Load Review Report

1. Find latest report in `.ai-context/reviews/` (or use specified path)
2. Parse report to extract issues
3. Filter issues by severity and category if specified
4. Display summary of issues to fix

### 2. Initialize Progress Tracking

```
TodoWrite:
- [ ] Fix N critical issues
- [ ] Fix N warnings
- [ ] Review N suggestions
```

### 3. Process Issues Interactively

For each issue (starting with Critical, then Warning, then Suggestion):

1. **Display Issue Context**

   ````
   ═══════════════════════════════════════════════════════════════
   Issue [1/15]: [BUG-001] Missing null check
   Severity: 🔴 Critical
   File: src/features/dashboard/application/hooks/useDashboard.ts:15
   ═══════════════════════════════════════════════════════════════

   Issue: Accessing `data.metrics` without checking if `data` exists

   Current Code:
   ```typescript
   const metrics = data.metrics;
   ````

   Suggested Fix:

   ```typescript
   const metrics = data?.metrics;
   ```

   Rule: [Bug Detection Standards](../../rules/code-review-standards.md#bug-detection-standards)

   ```

   ```

2. **Ask for Confirmation**

   ```
   Options:
   - [Y] Apply this fix
   - [N] Skip this issue
   - [M] Modify the fix before applying
   - [V] View more context
   - [A] Apply all remaining fixes
   - [Q] Quit fixing
   ```

3. **Apply Fix** (if confirmed)
   - Read the file
   - Apply the change using Edit tool
   - Mark issue as fixed in progress tracker

4. **Move to Next Issue**

### 4. Generate Fix Summary

After all issues processed:

```markdown
## Fix Summary

| Action   | Count |
| -------- | ----- |
| Fixed    | 12    |
| Skipped  | 3     |
| Modified | 2     |

### Fixed Issues

- [BUG-001] Missing null check - useDashboard.ts
- [ARCH-001] Cross-feature import - Header.tsx
- ...

### Skipped Issues

- [STYLE-003] Class ordering - Reason: Intentional for readability

### Files Modified

- src/features/dashboard/application/hooks/useDashboard.ts
- src/features/dashboard/presentation/components/Header.tsx
- ...
```

### 5. Suggest Verification

```
Fixes complete!

Recommended next steps:
1. Run `pnpm type-check` to verify no type errors
2. Run `pnpm test` to ensure tests pass
3. Run `/full-review` to verify issues are resolved
```

## Fix Strategies by Category

### Architecture (ARCH)

| Issue Type            | Fix Strategy                        |
| --------------------- | ----------------------------------- |
| Cross-feature import  | Move shared code to `@/shared/`     |
| Layer violation       | Restructure import to correct layer |
| Missing barrel export | Create `index.ts` with exports      |
| App Router logic      | Extract to feature page component   |

### SOLID Violations

| Issue Type    | Fix Strategy                         |
| ------------- | ------------------------------------ |
| SRP violation | Extract to separate components/hooks |
| OCP violation | Add prop variants or composition     |
| ISP violation | Split interface into smaller ones    |
| DIP violation | Introduce abstraction hook           |

### DRY Violations

| Issue Type       | Fix Strategy                   |
| ---------------- | ------------------------------ |
| Duplicate code   | Extract to shared utility      |
| Repeated pattern | Create reusable component/hook |
| Magic values     | Extract to constants file      |

### Bug Fixes (BUG)

| Issue Type      | Fix Strategy                 |
| --------------- | ---------------------------- |
| Null access     | Add optional chaining (`?.`) |
| Missing cleanup | Add return cleanup function  |
| Missing deps    | Add to dependency array      |
| Stale closure   | Use ref or add to deps       |
| Async error     | Wrap in try/catch            |

### Naming (NAME)

| Issue Type    | Fix Strategy                    |
| ------------- | ------------------------------- |
| File name     | Rename file with correct casing |
| Variable name | Use replace_all to rename       |
| Import order  | Reorder imports                 |

| Issue Type       | Fix Strategy            |
| ---------------- | ----------------------- |
| Hardcoded string | Replace with `t('key')` |
| Missing key      | Add key to locale files |

### Styling (STYLE)

| Issue Type         | Fix Strategy                |
| ------------------ | --------------------------- |
| Non-semantic color | Replace with semantic token |
| Missing cn()       | Wrap in cn() utility        |
| Class ordering     | Reorder classes             |

### Testing (TEST)

| Issue Type        | Fix Strategy                    |
| ----------------- | ------------------------------- |
| Missing test file | Create test file skeleton       |
| Wrong query       | Replace with user-centric query |

### Security (SEC)

| Issue Type     | Fix Strategy                 |
| -------------- | ---------------------------- |
| Exposed secret | Move to environment variable |
| XSS risk       | Add sanitization             |

### Performance (PERF)

| Issue Type   | Fix Strategy                |
| ------------ | --------------------------- |
| Missing memo | Wrap in useMemo/useCallback |
| Missing lazy | Add React.lazy wrapper      |

## Interactive Commands

During fixing, you can use these commands:

| Command    | Description                           |
| ---------- | ------------------------------------- |
| `Y` or `y` | Apply the suggested fix               |
| `N` or `n` | Skip this issue                       |
| `M` or `m` | Modify the fix before applying        |
| `V` or `v` | View more file context                |
| `A` or `a` | Apply all remaining fixes (auto mode) |
| `Q` or `q` | Quit fixing, show summary             |
| `?`        | Show help                             |

---

## Notes

- Always review fixes before applying
- Use `--dry-run` first for large reports
- Critical issues should be fixed before merge
- Some suggestions may be intentional - use Skip
- After fixing, run tests to verify nothing broke
